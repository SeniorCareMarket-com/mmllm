### Gap analysis

#### What did the residual target NOT fix?

1. **The 3000-step ceiling on stability.** Both my run (round9-0 comm) and possibly the round9-1 search_info run (which round9-1's narrative reports ctrl_bpc going noisier than round-8) suggest the residual-target + LR_BANK=10× + LR_NET-cosine + no-sleep configuration produces a working point that's productive but fragile. There's a numerical instability that fires around step 3000–4000 in at least one of the five workers.

2. **The format_validity inheritance problem.** Round9-1's narrative articulates this clearly: format_validity is a property of dense weight state, and any single-worker training drifts dense away from it. The residual-target architectural fix is orthogonal to this — making Net carry more signal doesn't defend dense from drifting.

3. **Cross-cycle decay was already gone in this run (no sleep), but cross-step decay within the clean regime still happened.** Δ_net went 0.077 → 0.027 over steps 250–1000 (cycle-1 decay), bounced, peaked again, declined again. The "Net learns a static target as dense absorbs Local" mechanism that round9-1's narrative hypothesizes is consistent with my data — between cycles the trajectory is mean-reverting, not strictly ascending.

#### What looks like a bug, not a tuning issue

The **distill=3.45, replay=2.85 router-metrics spike at ~step 3000** preceded the ctrl_bpc explosion by ~250 steps. That's not a normal mid-training fluctuation. Possibilities:

- A NaN/inf appearing in one of NetBank's V rows under SparseAdam (would cascade through `(local_out − sdpa_out)` via Net's output and into the distill gradient).
- An outlier batch with extreme sequence content that pushed Local's V into a high-magnitude regime, then dense couldn't keep up.
- A SparseAdam moment-state mismatch when one of the parallel param groups hits a step where its replay sample induces a zero-gradient update for many rows.

Whichever it is, **no sleep mechanism is involved** (it was removed). The trigger is in training, not in the sleep boundary. The smoking-gun signal in the telemetry is the replay loss going 0.27 → 2.85 (10×) in a single sample window. If the sparse-update path has a numerical guard, this run would have flagged it.

#### Mechanism interpretation

Pre-spike (steps 250–3000), this run **does** look like genuine durable consolidation by some honest metric:
- distill_loss sustained ≫ round-8 floor.
- Δ_net oscillating well above zero, not decaying to noise floor.
- Synergy went positive for the first time in any round (steps 1250–1750).

Post-spike, the run is broken and the high Δ_net values are an artifact.

The big-picture read: **residual-target distill is the right architectural move; the LR config that enabled it is unstable enough that 1-in-5 workers blows up.** Whether to call this "the architecture works, fix the training stability" vs "the architecture has a hidden numerical issue" depends on whether the other 4 round-9 re-runs blew up or not. I had time only for shard 0; the coordinator can check the other shards' ctrl trajectories for the same shape.

### One concrete fix to test next

**Add a `MMLLM_GRAD_CLIP_BANK` threshold at the SparseAdam step.** Currently AdamW dense gradients clip via `--grad-clip` (from the commit history), but the bank-tier sparse path has no equivalent clip. If the step-3000 spike was triggered by a high-norm V update on one or two rows that pushed `local_out` far from sdpa, then `(local_out − sdpa_out)` exploded and the distill MSE pulled Net's V in the same direction at coef=1.0, cascading the instability.

Telemetry that would tell you it worked:
- `net_v_grad_norm` and a new `local_v_grad_norm` (per the netbank_telemetry already in the log schema) should show 99th-percentile spikes around step 3000 in this run.
- After the clip is added, those percentiles should compress and ctrl_bpc should stay in 0.25–0.35 throughout.

If you don't see a v_grad_norm spike in this run's telemetry at all, it's not a sparse-update issue; it's something in the dense path, and a different fix is needed (likely just lowering LR_BANK_MULT from 10× to 5×).

### Honest summary

**The architectural fix moved the needle for the first time.** Steps 250–3000 of round9-0 are the cleanest "bank tier carries durable signal that dense+sdpa don't reproduce" result the project has produced. Δ_net mean ≈ +0.057 over 12 ablations, 10–40× the comparable values in rounds 7/8/9-2.

**The architectural fix did not fix the format_validity regression, and the run blew up mid-training.** The fim_eval came in 8.017 — worst single-worker number on record. End-of-run telemetry is confounded by the broken-dense regime.

The breakout is real and meaningful; the production-readiness gap is now in training stability, not architectural design. **Next dispatch should hold the residual-target distill and Bernoulli-gate config and focus on the numerical stability fix above before scaling.**
